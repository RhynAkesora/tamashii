## 1. 概要

「魂の共鳴 ～私を信じて～」は、ブロックパズルに砂粒子物理シミュレーションを組み合わせた
ユニークなパズルゲームです。テトロミノ（4ブロック）とペントミノ（5ブロック）を操作し、
同色の砂粒子で横一列を繋げて消していきます。

### コンセプト
- **砂の物理演算:** ブロックが着地すると砂粒子に分解され、自然落下・滑落する
- **5属性カラー:** 火（赤）・水（青）・風（緑）・地（金）・光（白）
- **コンボ＆フィーバー:** 連鎖で高スコアを狙い、フィーバーモードで2倍ボーナス
- **マインドフルモード:** スコアなし・ゲームオーバーなしの瞑想的プレイ
- **レトロ風ビジュアル:** SFC（スーパーファミコン）風の3Dブロック描画

## 4. ゲームメカニクス

### 4.1 グリッド構成

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| グリッド幅 | 9列 | ブロック単位 |
| グリッド高 | 19行 | ブロック単位 |
| 砂グリッド幅 | 45 (9×5) | 砂粒子単位 |
| 砂グリッド高 | 95 (19×5) | 砂粒子単位 |
| セル分割数 | 5×5 | 1ブロック = 25砂粒子 |

### 4.2 ピース（ブロック）

**7種類のピース**（テトロミノ4種 + ペントミノ3種）:

| ピース名 | ブロック数 | 形状 |
|----------|-----------|------|
| I | 4 | 直線（1×4） |
| O | 4 | 正方形（2×2） |
| L | 4 | L字型 |
| S | 4 | Z/S字型 |
| P | 5 | ペントミノ型 |
| T5 | 5 | 十字/プラス型 |
| U | 5 | U字/コの字型 |

### 4.3 カラーシステム

| 属性 | カラーコード | 解放レベル |
|------|-------------|-----------|
| 火 (Fire) | `#E8423F` 赤 | Lv1〜 |
| 水 (Water) | `#3B8ED0` 青 | Lv2〜 |
| 風 (Wind) | `#3CC76A` 緑 | Lv3〜 |
| 地 (Earth) | `#D4A017` 金 | Lv6〜 |
| 光 (Light) | `#F0E6D3` 白 | Lv10〜 |

### 4.4 ピース操作

| 操作 | 説明 |
|------|------|
| 左右移動 | 1セルずつ移動 |
| 右回転 | 時計回り（ウォールキック対応: [0, -1, 1, -2, 2]） |
| 左回転 | 反時計回り（同上） |
| ソフトドロップ | 長押し(300ms)で加速落下 |
| ハードドロップ | 即座に最下部へ |
| ホールド | 現在のピースを保管し、保管中のピースと入れ替え |

**ロック遅延:** 30フレーム（約0.5秒）、最大15操作まで延長可能

### 4.5 砂物理エンジン

ブロックが着地すると、各セルは5×5=25個の砂粒子に分解されます。

**物理ルール:**
1. 真下が空いていれば自然落下
2. 真下がブロックされていれば斜め下へ滑落（左右交互）
3. Set構造で衝突判定（O(1)探索）
4. フレームごとに1〜2回の物理演算イテレーション

### 4.6 ライン消去

**検出ロジック:**
1. 同色の砂粒子で連結成分を探索（DFS）
2. 連結成分が砂グリッド幅（45セル）を横断しているか判定
3. 横断していればライン消去（フラッシュアニメーション: 18フレーム）

### 4.7 スコアリング（クラシックモード）

**基本スコア:**
```
basePoints = 消去ライン数 × 100 × フィーバー倍率
```

**コンボボーナス:**
```
comboBonus = 100 × 1.5^(comboCount - 1) × フィーバー倍率
（comboCount > 1 の場合のみ）
```

**オールクリアボーナス:**
- 条件: 5コンボ以上でボード全消去
- 報酬: 3,000点（フィーバー中: 6,000点）

### 4.8 レベルシステム

```
レベル = ⌊累計消去ライン数 / 4⌋ + 1
落下速度倍率: fallSpeed × 0.975（4ライン消去ごと）
```

| レベル帯 | 背景テーマ | 使用色数 |
|---------|-----------|---------|
| Lv1〜3 | 砂漠（昼） | 1〜3色 |
| Lv4〜6 | 砂漠（夕暮れ） | 4色 |
| Lv7〜9 | 砂漠（夜） | 4色 |
| Lv10+ | 宇宙 | 5色（全色解放） |

### 4.9 フィーバーモード

| パラメータ | 値 |
|-----------|-----|
| 発動条件 | 3コンボ以上 |
| 基本時間 | 20秒（1,200フレーム） |
| ライン消去で延長 | +2秒/ライン |
| 最大時間 | 40秒（2,400フレーム） |
| スコア倍率 | 2倍 |
| 専用BGM | `fever-bgm.mp3` |

### 4.10 危険警告

| 段階 | 条件 | 表示 |
|------|------|------|
| 安全 | 上5行にブロックなし | なし |
| 警告 | 2〜4行目にブロック | 黄色グラデーション |
| 危険 | 0〜1行目にブロック | 赤グラデーション + 点滅枠 + "DANGER"テキスト |

## 5. ゲームモード

### 5.1 クラシックモード

標準的なスコアアタックモード。

- レベルに応じて使用色が増加
- コンボ3以上でフィーバー発動
- ブロックが最上段に達するとゲームオーバー
- ゲームオーバー時: スコア・消去ライン数・最大コンボ・レベル・プレイ時間を表示
- リワード広告視聴で1回復活可能（上4行を除去）

### 5.2 マインドフルモード

瞑想的・ストレスフリーなモード。

| 要素 | 仕様 |
|------|------|
| ゲームオーバー | なし（溢れた場合はリセットアニメーションで回復） |
| スコア | なし（消去ライン数のみ表示） |
| フィーバー | なし（代わりにコンボで穏やかな色相シフト） |
| 制限時間 | 5分 / 10分 / 15分から選択 |
| 初回チュートリアル | 4ページの哲学的解説 |
| 終了画面 | ライン数・リセット回数・最大コンボを表示 |
| BGM | 環境音（`soul-mindful.mp3`） |

## 6. ビジュアルシステム

### 6.1 背景テーマ（4段階）

#### テーマ0: 砂漠（昼）— Lv1〜3
- 空: 青〜オレンジのグラデーション
- 砂丘のシルエットと太陽の光芒
- カラー: 温かみのあるサンド＆ゴールド系

#### テーマ1: 砂漠（夕暮れ）— Lv4〜6
- 空: 燃えるようなオレンジ〜赤
- 地平線のオレンジグロー
- カラー: 深い赤＆ブラウン系

#### テーマ2: 砂漠（夜）— Lv7〜9
- 空: 深い紫〜黒
- 星空（25〜30個の固定星座配置）
- 月の微光（左側）

#### テーマ3: 宇宙 — Lv10+
- 宇宙的な深い青＆紫のグラデーション
- 星雲（放射状グラデーション2層）
- 天の川（対角線バンド）
- 星: 150個（明滅あり）+ 8個の十字光芒付き明星
- 惑星の輪郭（右下弧）

### 6.2 ブロック描画

- SFC風3Dエフェクト（上/左にハイライト、下/右にシャドウ）
- 中央に宝石風の輝き
- ゴーストピース: 着地位置をパルス半透明 + 白アウトラインで表示

### 6.3 エフェクト

| エフェクト | トリガー | 説明 |
|-----------|---------|------|
| 画面シェイク | ライン消去・コンボ・レベルアップ | 強度はイベントに比例（マインドフルでは30%に抑制） |
| フローティングテキスト | スコア加算・コンボ・レベルアップ | 色分け表示、上昇＋フェードアウト（60フレーム寿命） |
| パーティクル | 着地・ハードドロップ・ライン消去・レベルアップ | 放射状パーティクル |
| フィーバーオーバーレイ | フィーバー中 | マゼンタ脈動・4本のライトビーム・8個の浮遊粒子 |
| ヨギグロー | コンボ4+ | 金色ハロー＋回転する光の点 |

### 6.4 ヨギキャラクター

右サイドパネルに表示されるキャラクター。コンボに応じて輝度が変化。

| コンボ数 | 透明度 | 効果 |
|---------|--------|------|
| 0 | 25% | 暗い |
| 1 | 35% | やや明るい |
| 2 | 45% | 中程度 |
| 3 | 55% | 明るい |
| 4+ | 70% | 金色ハロー + 白色ハロー + 回転光点 |

### 9.2 Firebase Firestore

**コレクション: `scores`**

```json
{
  "name": "プレイヤー名（最大10文字、サニタイズ済）",
  "score": 12500,
  "maxCombo": 8,
  "createdAt": "Timestamp（サーバー側生成）"

```

**クエリ:** `ORDER BY score DESC LIMIT 20`

### 9.3 ハイスコア名前入力

#### 入力UI

- HTML `<input type="text">` による名前入力モーダル（Canvas上にオーバーレイ表示）
- `maxlength="10"` / `z-index: 30`
- `position: fixed` + `align-items: flex-start` + `padding-top: 15vh` でキーボード表示時も入力欄が見える配置
- モーダル内フォーム: `max-width: 360px; width: 90%`（タブレット等大画面でも適切なサイズ）
- 「登録」ボタンと「スキップ」ボタンの2択（スキップ時は匿名で登録）

#### 表示条件

- ゲームオーバー時にスコアが Top 20 入りしている場合に表示
- 復活ボタン使用後（リトライ/タイトルへ遷移時）にもハイスコア判定を実施
- Firebase未接続時は名前入力をスキップ

#### 名前サニタイズ（`sanitizePlayerName()`）

スコア登録時の名前入力に対して以下のフィルタリングを実施:

1. トリム & 10文字制限
2. 制御文字・ゼロ幅Unicode除去（`\u0000-\u001F`, `\u200B-\u200F` 等）
3. 全角→半角変換（`\uFF01-\uFF5E` → ASCII）
4. カタカナ→ひらがな変換（`\u30A1-\u30F6` → ひらがな）
5. 句読点・スペース・記号除去
6. リートスピーク変換（`0→o, 1→i, 3→e, 4→a, 5→s, 7→t, 8→b, $→s, @→a`）
7. NGワードリスト照合（後述）
8. 未入力時デフォルト名: 「名無しの魂」(ja) / "Nameless Soul" (en)

#### NGワードリスト

NGワード判定は正規化後の文字列に対して行われる（大文字小文字・全角半角・リートスピーク回避を無効化）。

**部分一致ブロック**（名前の一部に含まれていれば拒否）:

| カテゴリ | ワード |
|---------|--------|
| 英語 侮辱・性的 | `fuck`, `fck`, `fuq`, `shit`, `sht`, `dick`, `pussy`, `bitch`, `btch`, `cunt`, `penis`, `porn`, `hentai` |
| 英語 差別 | `nigger`, `nigga`, `nigg`, `fagg`, `rape`, `nazi`, `hitler` |
| 日本語 暴力 | `しね`, `ころす`, `ころせ`, `くたばれ`, `死ね`, `殺す`, `殺せ` |
| 日本語 下品 | `くそ`, `うんこ`, `ちんこ`, `まんこ`, `おっぱい`, `糞` |
| 日本語 差別 | `きちがい`, `基地外`, `障害者`, `ガイジ` |

**完全一致ブロック**（短い単語の誤検知防止のため、名前全体が一致した場合のみ拒否）:

`ass`, `sex`, `die`, `kill`, `fag`

※ マッチした場合はデフォルト名（「名無しの魂」/ "Nameless Soul"）に置換される

## 12. プラットフォーム対応

### 12.1 PWA (Web)

- Service Worker によるオフライン完全対応

- App Tracking Transparency (ATT) ダイアログ実装
- ステータスバー: 黒半透明スタイル

- 最小対応: Android 5.0+ (API 21+)
- AAB形式での提出対応
- 64-bit アーキテクチャ対応
- 戻るボタン適切処理

### 12.4 レスポンシブデザイン

- 動的キャンバスサイズ算出（デバイスに応じてスケーリング）
- 小画面閾値: 600px（フォントサイズ・スペーシング調整）
- Safe Area Insets のCSS変数活用
- Retinaディスプレイ対応（デバイスピクセル比反映）
