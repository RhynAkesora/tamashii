# プロジェクトルール

## Git操作
- push、pull、commit、merge などの git 操作は、できる限りアシスタント側で自主的に実行すること
- ユーザーに git 操作を委ねず、作業完了時に自動的に commit & push まで行う

## バージョン管理
- index.html を1行でも変更したら、バージョンを 0.001 上げること（確認不要）
- バージョン表記は index.html 内の `ctx.fillText('Ver. X.XXX', ...)` を更新する
- バージョン表記はタイトル画面と設定画面の2箇所に存在するため、更新時は必ず両方を同時に更新すること
- service-worker.js の `CACHE_NAME` もバージョンに合わせて更新すること（例: `tamashii-no-kyoumei-v3724`）
- 応答の末尾に現在のバージョンを表記すること（例: `現在のバージョン: 3.720`）

## リリース準備作業（段階的対応）
- 正式リリースに向けた全タスクは `RELEASE_CHECKLIST.md` に記載されている
- コンテキスト制限があるため、1セッションで全タスクを完了する必要はない
- 各セッションではフェーズ番号順に、1〜2フェーズ分のタスクを対応する
- タスク完了時は `RELEASE_CHECKLIST.md` のチェックボックスを `[x]` に更新する
- セッション開始時に `RELEASE_CHECKLIST.md` を読み、未完了タスクの続きから着手する
- 作業前に必ず対象コードの現在の行番号を確認すること（行番号は変更により変動する）
- フェーズ1（クリティカルバグ）→ フェーズ2（中優先度バグ）→ … の順で進める
